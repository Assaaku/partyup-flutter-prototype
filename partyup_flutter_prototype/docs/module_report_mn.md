# PartyUp frontend модулын тайлангийн богино хэсэг

## 1. Хөгжүүлэх модулын тодорхойлолт, шаардлагууд

Энэхүү prototype нь PartyUp системийн хэрэглэгчийн интерфейсийн үндсэн модулиудыг харуулна. Үүнд бүртгэлгүй хэрэглэгчийн эвент харах хэсэг, утасны дугаартай нэвтрэлт, админ нэвтрэлт, эвент feed, газрын зураг, эвент үүсгэх, чат, профайл, админ moderation panel багтсан.

Хэрэглэгчийн талд ойролцоох эвентүүдийг харах, ангиллаар шүүх, эвентэд нэгдэх, хадгалах, дэлгэрэнгүй мэдээлэл харах боломжийг mock data ашиглан үзүүлэв. Байгууллага болон хувь хүн зохион байгуулагчийн эвентүүдийг нэг feed дотор харуулж, жижиг бүлгийн уулзалт болон том байгууллагын арга хэмжээг ялгаж тэмдэглэсэн.

## 2. Ашигласан технологи

- Flutter / Dart
- Material 3 UI
- Mock service layer
- Local in-memory state
- Asset image: `assets/images/map.png`

## 3. Системийн үйл ажиллагаа

1. Бүртгэлгүй хэрэглэгч эвентүүдийг харах болон газрын зураг үзэх боломжтой.
2. Хэрэглэгч `+976 ********` форматаар утсаа оруулж нэвтрэх боломжтой.
3. Админ `admin / pass` ашиглан админ самбар руу орно.
4. Нэвтэрсэн хэрэглэгч эвентэд нэгдэх, хадгалах, эвент үүсгэх боломжтой.
5. Админ хэрэглэгч, байгууллага, эвент, санал хүсэлт, системийн аюулгүй байдлын mock хяналтын хэсгийг харна.

## 4. Аюулгүй байдлын тактикууд

Prototype түвшинд дараах тактикуудыг frontend дээр хэрэгжүүлсэн.

- Input validation: утасны дугаар 8 оронтой эсэх, насны шалгалт, хоосон талбар шалгах.
- Input sanitization: эвент үүсгэх үед хэрэглэгчийн оруулсан текстээс аюултай тэмдэгтүүдийг цэвэрлэх.
- Role-Based Access Control: зөвхөн admin role-той хэрэглэгч админ самбар харах боломжтой.
- Rate limiting: админ нэвтрэлтийн буруу оролдлогыг хязгаарлах mock хамгаалалт.
- Audit log: login, logout, event join, moderation зэрэг үйлдлийг бүртгэх.
- Privacy-aware error: нэвтрэлтийн үед яг аль талбар буруу гэдгийг ил тод хэлэхгүй ерөнхий алдаа харуулах.

## 5. Дүгнэлт

Энэ prototype нь PartyUp системийн үндсэн frontend урсгал, хэрэглэгчийн оролцоо, байгууллагын эвент, админ хяналт болон аюулгүй байдлын суурь тактикуудыг харуулахад зориулагдсан. Backend холболт хийгдээгүй боловч service layer-ийг тусгаарласан тул Firebase, REST API, JWT authentication зэрэг бодит системтэй холбоход бэлэн бүтэцтэй.
